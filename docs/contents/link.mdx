## API Reference

### Link

A component that handles both internal navigation and external URL opening with smart browser behavior.

| Prop       | Type            | Default | Description                                                |
| ---------- | --------------- | ------- | ---------------------------------------------------------- |
| `href`     | `Href & string` | -       | The URL or route to navigate to.                           |
| `children` | `ReactNode`     | -       | The content to display inside the link.                    |
| `external` | `boolean`       | `false` | Whether to open external URLs in system browser vs in-app. |
| `onPress`  | `function`      | -       | Custom press handler (extends default behavior).           |

### Additional Props

The Link component accepts all props from Expo Router's `Link` component except `href` (which is required and typed differently).

## Behavior

### Internal Navigation

- Uses Expo Router for internal app navigation
- Supports all Expo Router href formats (strings, objects)
- Maintains navigation history and state
- Works consistently across platforms

### External URLs

The component intelligently handles external URLs based on platform and the `external` prop:

**Native Platforms (iOS/Android):**

- `external={false}` (default): Opens in in-app browser via `expo-web-browser`
- `external={true}`: Opens in system browser via `Linking.openURL`

**Web Platform:**

- Always opens in new tab/window regardless of `external` prop
- Uses standard web link behavior

### Platform Differences

- **Native**: Prevents default browser opening, uses custom handlers
- **Web**: Uses standard web link behavior with `target="_blank"`
- **All platforms**: Internal navigation works consistently

## URL Types

### Internal Routes

```tsx
<Link href="/profile">Profile</Link>
<Link href="/settings/account">Account Settings</Link>
<Link href="/(tabs)/home">Home Tab</Link>
<Link href={{ pathname: '/user/[id]', params: { id: '123' } }}>User 123</Link>
```

### External URLs

```tsx
{/* Opens in in-app browser (default on native) */}
<Link href="https://example.com">Website</Link>

{/* Opens in system browser */}
<Link href="https://example.com" external>Website</Link>

{/* Special URL schemes */}
<Link href="mailto:hello@example.com">Email</Link>
<Link href="tel:+1234567890">Phone</Link>
```

### Dynamic Routes

```tsx
<Link href={`/user/${userId}`}>User Profile</Link>
<Link href={{ pathname: '/search', params: { q: searchTerm } }}>
  Search Results
</Link>
```

## Content Types

### Text Links

When `children` is a string, it automatically wraps with styled Text component:

```tsx
<Link href='/about'>About Us</Link>;
{
  /* Renders: <Text variant="link">About Us</Text> */
}
```

### Custom Content

For custom content, provide any React elements:

```tsx
<Link href='/profile'>
  <View style={styles.customLink}>
    <Icon name='user' />
    <Text>My Profile</Text>
  </View>
</Link>
```

## Styling

### Text Styling

The component automatically applies the `link` text variant for string children. Customize this in your Text component's variant system.

### Custom Styling

Apply additional styles using standard React Native style props:

```tsx
<Link href='/home' style={{ padding: 10, backgroundColor: 'blue' }}>
  Styled Link
</Link>
```

## Error Handling

The component includes basic error handling for URL operations:

- Wraps `Linking.openURL` and `openBrowserAsync` in try-catch blocks
- Gracefully falls back if browser opening fails
- Maintains app stability during navigation errors

## Accessibility

The Link component maintains full accessibility support:

- Preserves all accessibility props from Expo Router
- Supports screen reader navigation and announcements
- Maintains proper focus behavior across platforms
- Works with keyboard navigation where supported
- Announces link destinations to assistive technologies

## Best Practices

### URL Handling

- Use `external={true}` for URLs that should open in system browser
- Test external link behavior on both platforms
- Consider user preferences for in-app vs system browsing

### Content

- Use descriptive link text that makes sense out of context
- Distinguish between internal and external links visually
- Provide loading feedback for slow external links

### Performance

- Avoid creating new objects in href prop on every render
- Pre-load critical navigation routes when possible
- Consider lazy loading for heavy external content

### Testing

- Test all link types on both iOS and Android
- Verify external links work with various URL schemes
- Test navigation state preservation
- Ensure proper back button behavior
